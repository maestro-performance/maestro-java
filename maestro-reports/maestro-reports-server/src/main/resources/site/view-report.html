The latencies displayed in the table above are show in microseconds whereas the chart displays them in milliseconds.
Tests without a fixed rate will not display response time metrics. That is expected when running all-out tests because the backend is unable to estimate the response time based on the rate. Ideally, Maestro performance tests should use fixed rates and evaluate the pass/fail condition based on the (service time) latency at the 90th percentile.

Rate Distribution Over Time
